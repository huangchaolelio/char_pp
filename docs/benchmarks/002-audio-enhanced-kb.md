## 实际视频验证计划（SC-001 / SC-002 / SC-005）

### 验证数据集要求

- **SC-001 验证集**: 10 段含明确口头技术讲解的乒乓球教学视频（每段 2-5 分钟），人工标注每段中语音提及的技术要点数量（含维度名称和参数值）
- **SC-002 验证集**: 5 段含教练明确口头示范提示词的视频，人工标注高峰帧所在时间段
- **SC-005 验证集**: 抽取 20 条系统生成的技术要点（含参数描述），与对应转录原文对比

### 验证步骤

```bash
# SC-001: 音频→KB 贡献比
# 1. 对验证集视频逐一运行完整管道
# 2. 统计每段 source_type="audio" 或 "visual+audio" 的技术要点数
# 3. 计算: audio_points / total_points >= 0.50

# SC-002: 片段命中率
# 1. 对同一视频分别运行纯视觉模式 (enable_audio_analysis=false) 和音频辅助模式
# 2. 统计各模式提取片段中含姿态峰值帧的比例
# 3. 验证: audio_peak_rate - visual_peak_rate >= 0.30

# SC-005: 技术要点准确率
# 1. 从已处理视频中随机抽取 20 条 source_type="audio" 技术要点
# 2. 与 AudioTranscript.sentences 原文对比
# 3. 验证: 无歧义偏差条目数 >= 16
```
