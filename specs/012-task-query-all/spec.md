# 功能规范: 全量任务查询接口

**功能分支**: `012-task-query-all`
**创建时间**: 2026-04-23
**状态**: 草稿
**输入**: 用户描述: "增加全量的任务查询接口,返回所有任务相关信息"

## 用户场景与测试 *(必填)*

### 用户故事 1 - 管理员查看全部任务列表 (优先级: P1)

运营/管理人员需要一个接口，能够一次性获取系统中的所有任务记录，查看每个任务的基本信息、处理状态、进度和时间戳，以便监控整体处理情况和排查问题。

**优先级原因**: 这是全量查询的核心功能，其他筛选和关联信息都建立在列表查询的基础上，P1 优先交付能立即提供业务价值。

**独立测试**: 可以通过请求接口获取任务列表，检查返回的任务条数和字段完整性，验证分页和排序是否正常，无需任何其他故事完成。

**验收场景**:

1. **给定** 系统中存在若干任务记录（含不同状态），**当** 请求全量任务列表接口时，**那么** 返回所有未被物理删除的任务，每条记录包含任务 ID、类型、状态、视频文件名、进度百分比、创建时间、开始时间、完成时间。
2. **给定** 请求带有分页参数（page=2, page_size=20），**当** 调用接口，**那么** 只返回第 2 页的最多 20 条记录，响应中同时包含总记录数和总页数。
3. **给定** 请求带有状态筛选参数（status=failed），**当** 调用接口，**那么** 只返回状态为 failed 的任务。
4. **给定** 系统中没有任何任务，**当** 请求接口，**那么** 返回空列表和总数为 0，不报错。

---

### 用户故事 2 - 查询单个任务的完整关联信息 (优先级: P2)

开发/运营人员排查某一具体任务时，需要在一次请求中获取该任务的所有关联数据——包括任务本身、关联教练、提取的技术点摘要、运动员动作分析摘要、音频转录状态等，避免多次请求拼接数据。通过扩展现有 `/tasks/{task_id}` 接口响应体实现，无需调用方切换端点。

**优先级原因**: 列表接口（P1）只返回基础字段，关联信息的完整聚合是独立的需求切片，优先级略低但对问题排查价值很高。

**独立测试**: 可以通过传入任意已知任务 ID，检查响应中是否包含所有关联实体的摘要字段，独立于列表查询接口。

**验收场景**:

1. **给定** 一个存在的教练视频任务 ID，**当** 请求该任务的完整信息，**那么** 响应除基础字段外，还包含：关联教练姓名（若有）、提取的技术点数量、音频转录是否存在、语义分段数量、处理耗时统计。
2. **给定** 一个存在的运动员视频任务 ID，**当** 请求该任务的完整信息，**那么** 响应还包含：动作分析分段数量、偏差报告数量、建议条数。
3. **给定** 一个不存在的任务 ID，**当** 请求完整信息，**那么** 返回明确的"任务不存在"提示，而非系统错误。
4. **给定** 一个已软删除的任务 ID，**当** 请求完整信息，**那么** 返回"任务不存在"提示，与正常未找到场景行为一致。

---

### 用户故事 3 - 按多维度筛选和排序任务 (优先级: P3)

开发人员调试或运营人员统计时，希望能按任务类型、状态、时间范围、教练 ID 等条件组合筛选，并按创建时间或完成时间排序，快速定位目标任务集合。

**优先级原因**: 筛选和排序是对列表查询的增强，在任务量大时才能体现价值，优先级低于基础列表和详情。

**独立测试**: 可以分别传入不同筛选参数组合请求列表，验证每种筛选条件的正确性，与其他故事独立。

**验收场景**:

1. **给定** 同时传入 task_type=expert_video 和 time_range（最近 7 天），**当** 请求列表，**那么** 只返回 7 天内创建的教练视频任务。
2. **给定** 传入 coach_id 参数，**当** 请求列表，**那么** 只返回该教练关联的任务。
3. **给定** 传入 sort_by=completed_at&order=desc，**当** 请求列表，**那么** 返回任务按完成时间由新到旧排列，completed_at 为 NULL 的未完成任务统一排在最末尾（NULLS LAST）。

---

### 边界情况

- 当 page_size 超过系统允许的最大值（如 200）时，自动截断为最大值并在响应中注明。
- 当传入无效的 status 枚举值时，返回明确的参数校验错误，不降级为返回全部任务。
- 当任务的关联实体（如 coach）已被删除时，关联字段返回 null，不影响任务记录本身的展示。
- 当同时传入多个筛选条件时，条件之间为 AND 关系。

## 需求 *(必填)*

### 功能需求

- **FR-001**: 系统必须提供一个列表查询接口，能够返回所有未被物理删除的任务记录，无默认时间范围限制，通过分页参数控制每次返回数量。
- **FR-002**: 列表接口必须支持分页查询，调用方可指定页码和每页数量，响应中包含总记录数和总页数。
- **FR-003**: 列表接口必须支持按任务状态（pending/processing/success/partial_success/failed/rejected）筛选。
- **FR-004**: 列表接口必须支持按任务类型（expert_video/athlete_video）筛选。
- **FR-005**: 列表接口必须支持按创建时间范围筛选（起始时间、结束时间均可选）。
- **FR-006**: 列表接口必须支持按教练 ID 筛选，仅返回该教练的相关任务。
- **FR-007**: 列表接口必须支持排序，可按创建时间或完成时间升降序排列，默认按创建时间倒序。按 completed_at 排序时，completed_at 为 NULL 的任务（未完成）统一排在末尾（NULLS LAST），无论升降序。
- **FR-008**: 列表接口每条任务记录至少包含：任务 ID、任务类型、状态、视频文件名、视频存储路径（video_storage_uri）、视频时长、进度百分比、错误信息（若有）、创建时间、开始时间、完成时间、关联教练 ID 和姓名（若有）。
- **FR-009**: 扩展现有单任务状态查询接口（`/tasks/{task_id}`），在原有基础字段之上额外返回所有关联实体的摘要统计：技术点数、转录是否存在、语义分段数、动作分析分段数、偏差报告数、建议条数、处理耗时统计。现有 `/tasks/{task_id}/result` 接口保持不变。
- **FR-010**: 对已软删除（deleted_at 不为空）的任务，两个接口均不予返回。
- **FR-011**: 当 page_size 超过系统最大限制时，自动截断为最大值（200 条），并在响应元数据中说明实际使用的 page_size。
- **FR-012**: 对非法参数（无效枚举值、负数分页等），接口必须返回参数校验错误，包含具体的错误字段和原因。

### 关键实体

- **任务（Task）**: 系统异步处理的单次视频分析请求，含类型、状态、视频元数据、处理进度、耗时统计。
- **关联统计摘要（Task Summary）**: 不含原始数据的聚合数字，包括技术点数量、分析分段数量、建议条数等，用于快速概览而无需深入子表。

## 成功标准 *(必填)*

### 可衡量的结果

- **SC-001**: 列表接口在系统存在 10,000 条任务记录时，单次查询（page_size=50）能在 1 秒内返回结果。
- **SC-002**: 单任务完整信息接口在任务含最多关联数据时，能在 2 秒内返回完整响应。
- **SC-003**: 所有查询参数的校验覆盖率达到 100%，非法参数均返回有意义的错误提示，不触发服务端异常。
- **SC-004**: 列表接口返回的分页元数据（总数、总页数）与实际数据完全一致，误差为 0。

## Clarifications

### Session 2026-04-23

- Q: 全量任务查询接口的访问控制级别是什么？ → A: 无额外鉴权，仅内网/服务间调用，不暴露到外网。
- Q: 列表和详情接口是否返回视频存储路径（video_storage_uri）？ → A: 返回原始存储路径，调用方可直接用于访问视频。
- Q: 单任务完整信息接口与现有 `/tasks/{task_id}` 和 `/tasks/{task_id}/result` 是什么关系？ → A: 扩展现有 `/tasks/{task_id}`，在响应体中增加关联统计摘要字段，不新增端点。
- Q: 列表接口不带筛选时，是否有默认时间范围限制？ → A: 无默认时间限制，仅靠分页（page/page_size）控制每次返回数据量。
- Q: 按 completed_at 排序时，尚未完成（completed_at 为 NULL）的任务排在哪里？ → A: NULL 排末尾（NULLS LAST），已完成任务优先展示。

## 假设

- 调用方为内部系统（服务间调用），接口不增加额外鉴权层，依赖内网隔离保障安全，不对外网暴露。
- 全量查询的"全量"指未软删除的任务，物理删除（deleted_at 不为空）的任务不在范围内。
- 关联统计摘要（技术点数、分析分段数等）通过聚合查询获取，不返回子表的具体记录内容。
- 接口挂载在现有 `/api/v1/tasks` 路由前缀下，保持与现有任务接口风格一致。
- 单次请求最大返回 200 条，超出时自动截断，不提供游标分页（cursor-based pagination）。
- 教练姓名通过 JOIN coaches 表获得，若教练记录已删除则返回 null，不影响任务本身展示。
