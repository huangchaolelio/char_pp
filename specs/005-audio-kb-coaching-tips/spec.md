# 功能规范: 音频技术要点提炼与教学建议知识库

**功能分支**: `005-audio-kb-coaching-tips`
**创建时间**: 2026-04-20
**状态**: 草稿
**输入**: 用户描述: "优化002,从音频中提取文字,总结讲解中该项技术的要点,形成知识库补充,方便为后续非专业运动员提供改进建议"

## 背景

Feature-002 已实现从教练视频音频中转录文字（Whisper），并尝试提取结构化技术参数（如数值范围）。但转录内容中还包含大量**非数值的教学描述**，例如：
- "击球时重心要主动前迎，不要等球"
- "引拍阶段保持放松，不要提前发力"
- "拍面角度稍微前倾，制造摩擦而不是撞击"

这些描述无法被当前的数值参数提取器识别，但对非专业运动员的纠错改进极具价值。本功能在现有音频转录基础上，增加**教学要点摘要**层，将教练口头讲解的核心技术建议提炼为结构化知识库条目，作为纯视觉数值参数的补充，共同构成更完整的指导建议来源。

---

## 用户场景与测试 *(必填)*

### 用户故事 1 - 从教练讲解中提炼技术要点描述 (优先级: P1)

管理员处理一段正手攻球教学视频。系统除了提取姿态数值（如肘部角度 127°）之外，还从教练的语音讲解中总结出该技术的关键要点描述，例如"击球瞬间手腕要有爆发性摩擦"、"引拍不能过长，保持紧凑发力"。这些要点作为文字性建议条目写入知识库，与数值参数条目并列，供后续为运动员生成改进建议时调用。

**优先级原因**: 当前知识库只有数值参数，对"为什么这样做"、"常见错误是什么"、"应该有什么感觉"等指导性描述覆盖为零，而这些恰恰是非专业运动员最需要听到的建议。

**独立测试**: 提交第06节正手攻球视频，查看知识库，验证除 4 条数值技术点外，还存在至少 2 条来源为"音频摘要"的文字要点，要点内容与教练讲解主题相关。

**验收场景**:

1. **给定** 一段教练讲解音频转录已完成，**当** 系统提炼要点，**那么** 知识库中出现至少 1 条类型为"教学建议"的条目，包含技术描述文本、对应动作类型和来源标注。
2. **给定** 教练在讲解中提到多个不同方面（引拍、击球、随挥），**当** 系统处理完成，**那么** 提炼出的要点按技术阶段分组，而不是混为一条。
3. **给定** 音频转录质量较差（识别词错误率高），**当** 系统无法提炼有意义的要点，**那么** 该视频的教学建议条目为空，不生成低质量内容，结果中注明"音频质量不足，建议人工补充"。

---

### 用户故事 2 - 教学建议在运动员指导中被调用 (优先级: P1)

教练提交运动员视频进行偏差分析后，系统在生成改进建议时，除了基于数值偏差给出"你的肘部角度偏低 15°"之外，还能附上来自知识库教学建议层的文字指导，如"引拍阶段保持放松，不要提前发力——这是孙教练第06节正手攻球课的要点"。改进建议同时包含量化偏差和定性改进方向，让非专业运动员既知道"偏了多少"，也知道"该怎么做"。

**优先级原因**: 仅告知数值偏差对非专业运动员不够直观，缺乏可操作的改进路径；文字建议补充后，指导完整性显著提升，运动员自主纠错能力增强。

**独立测试**: 提交一段正手攻球运动员视频，获取偏差分析结果，验证 coaching_advice 中至少包含 1 条附有知识库教学建议文字的条目，且来源可追溯到具体教练视频。

**验收场景**:

1. **给定** 运动员正手动作存在偏差，知识库中有对应动作类型的教学建议条目，**当** 系统生成改进建议，**那么** 建议内容包含"量化偏差说明"和"定性改进描述"两部分。
2. **给定** 知识库中有多条同一动作类型的教学建议，**当** 系统生成建议，**那么** 按 action_type 宽匹配附加最多 3 条（可配置）最新的人工审核或高置信度 TeachingTip，不堆砌全部条目。
3. **给定** 知识库中无对应动作类型的教学建议，**当** 系统生成建议，**那么** 仅提供数值偏差描述，不因缺少文字建议而报错。

---

### 用户故事 3 - 人工审核与编辑教学建议 (优先级: P2)

管理员可以查看系统自动提炼的教学建议条目，并对其进行编辑或删除。例如将 AI 总结不准确的描述修改为更精准的表达，或补充 AI 未捕捉到的关键要点。编辑后的条目被标记为"人工审核"，在后续运动员指导中优先于自动提炼条目使用。

**优先级原因**: AI 摘要存在理解偏差或表达不准确的风险，人工审核机制保证知识库质量，同时让教练可以持续完善知识库。

**独立测试**: 通过管理接口编辑一条自动提炼的教学建议，验证编辑后标记变为"人工审核"，再次生成运动员改进建议时使用的是编辑后版本。

**验收场景**:

1. **给定** 系统自动生成一条教学建议，**当** 管理员通过接口修改内容，**那么** 该条目标记变为"人工审核"，原始 AI 生成内容保留在历史记录中。
2. **给定** 同一技术要点存在"人工审核"和"自动提炼"两个版本，**当** 系统调用，**那么** 使用"人工审核"版本，"自动提炼"版本作为参考保留。

---

### 边界情况

- 当教练视频无音频或音频全程静默时，教学建议层为空，不影响数值技术点的正常提取。
- 当音频内容为纯示范（无口头讲解，如只有击球声）时，系统识别到无实质性讲解内容，教学建议层为空，并在结果中标注"无讲解内容"。
- 当同一技术阶段（如击球瞬间）提炼出多条要点描述存在矛盾时，系统保留全部并标注，由管理员审核。
- 当转录文本语言为非中文时，系统仍尝试提炼（支持英文），但置信度降低，结果标注"非标准语言，建议人工复核"。

## 需求 *(必填)*

### 功能需求

- **FR-001**: 系统必须在完成音频转录（Whisper）后，对转录文本进行技术要点摘要，提炼出该视频技术的核心教学建议，作为独立的知识库条目类型存储。
- **FR-002**: 每条教学建议条目必须包含：技术描述文本、对应动作类型（action_type）、技术阶段（如引拍/击球/随挥）、置信度、来源视频标识、来源类型（"自动提炼"）。
- **FR-003**: 系统必须根据转录文本中的技术阶段语义将要点分组，不将不同阶段的要点合并为单条。
- **FR-004**: 系统在提炼前必须先由 LLM 判断转录文本是否包含技术性讲解内容；判断为"无技术讲解"时跳过提炼，结果标注"无技术讲解内容"，不生成任何 TeachingTip 条目。
- **FR-005**: 系统在生成运动员改进建议时，必须在对应偏差维度的建议中附加来自知识库的教学建议文字（如存在匹配条目）。
- **FR-006**: 管理员必须能够通过 API 查看、编辑、删除教学建议条目；编辑后条目类型变更为"人工审核"。
- **FR-007**: 人工审核条目必须在运动员建议生成时优先于自动提炼条目使用。
- **FR-008**: 系统必须提供 `POST /tasks/{task_id}/extract-tips` 接口，对已完成的 expert_video 任务重新触发教学建议提炼，复用已有音频转录结果，不重新执行姿态分析和音频转录。重新提炼时，auto 状态的旧条目被替换，human 审核状态的条目保留不覆盖。

### 关键实体

- **TeachingTip（教学建议条目）**: 从音频转录摘要提炼的单条技术要点描述，包含文本内容、动作类型、技术阶段、置信度、来源视频、审核状态（auto/human）、创建时间。与 ExpertTechPoint 并列，共同构成知识库。**独立于 KB 版本审批流**，提炼完成后直接对运动员建议可用；human 审核状态条目优先于 auto 条目。
- **TechPhase（技术阶段）**: 动作的分解阶段标签，如"引拍"、"击球"、"随挥"、"站位"、"步法"；用于对教学建议分组，便于运动员按阶段查看改进方向。

## 成功标准 *(必填)*

### 可衡量的结果

- **SC-001**: 对已有音频转录的教练视频（第06节正手攻球，216句转录），系统能够提炼出 ≥3 条教学建议条目，内容与教练讲解主题一致，可被非专业运动员理解。
- **SC-002**: 运动员偏差分析报告中，存在知识库教学建议的动作类型，其改进建议内容包含文字指导的比例 ≥ 80%（不再是纯数值）。
- **SC-003**: 管理员通过接口编辑一条教学建议后，下一次运动员分析使用的是编辑后版本，人工审核条目优先级验证可 100% 通过。
- **SC-004**: 对无口头讲解内容的视频（纯动作示范），系统不生成任何教学建议条目，准确率 100%（不产生噪音）。
- **SC-005**: 教学建议提炼在已有转录的基础上，单次处理时间 ≤ 30 秒（不含重新转录时间）。

## 澄清

### Session 2026-04-20

- Q: 运动员改进建议中附加 TeachingTip 的匹配方式是什么？ → A: 按 action_type 宽匹配，偏差的 action_type 与 TeachingTip 相同即附加，每次建议最多附加条数可配置（默认 3 条）。
- Q: TeachingTip 是否需要经过 KB 版本审批流才能被运动员建议调用？ → A: 独立于 KB 版本，提炼完成后直接可用；审核状态（auto/human）决定优先级，不走版本审批流。
- Q: FR-004 跳过提炼的判断方式是什么？ → A: 由 LLM 先判断转录文本是否包含技术性讲解内容，有则提炼，无则跳过并标注"无技术讲解内容"。不依赖 Whisper 置信度分数预判。
- Q: FR-008 重新触发提炼的 API 粒度是什么？ → A: 按 task_id 触发，`POST /tasks/{task_id}/extract-tips`，复用该任务已有的转录结果重新提炼，不重新做姿态分析和音频转录。
- Q: "管理员"与"教练"角色是否区分权限？ → A: 不区分，所有 API 调用者权限相同，两者仅为叙述性描述，不引入认证授权控制。

## 假设

- 音频转录（Whisper）已由 Feature-002 实现，本功能在现有转录结果基础上增加摘要层，不重新实现转录。
- 要点提炼使用 LLM（与现有技术参数提取使用相同的模型接口），通过 prompt 工程控制输出格式。
- 当前知识库以正手/反手技术为主，教学建议的动作类型范围与现有 ActionType 枚举一致。
- 非专业运动员使用中文，教学建议以中文输出为主；英文音频的支持为次要目标。
- 教学建议条目不替换现有数值参数条目，两者并行存在，运动员建议生成时结合使用。
- 现有运动员建议生成逻辑（CoachingAdvice）已有基础结构，本功能扩展其内容来源，不重写整体流程。
- 系统无用户认证/授权体系，"管理员"和"教练"仅为叙述性角色描述，所有 API 调用者权限相同。
